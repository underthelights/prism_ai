Workflow

Move from real robot data collection to policy training and on-hardware validation inside one connected product flow.

Collect

Capture robot data with context

Capture demos on real hardware together with multi-camera context, TF state, and outcome labels.

RecordEpisode capture with success and failure annotation.
DevicesPre-flight checks for cameras and robot connectivity.
Data ToolsClean, merge, and validate collected datasets.
Train

Move from dataset to policy

Train IL and VLA policies while monitoring logs, GPU state, and experiment progress.

TrainingRun ACT, Diffusion, pi0.5, SmolVLA, GR00T, and related policies.
AugmentExpand vision and language data for better generalization.
EmbeddingInterpret experiments through latent-space analysis.
Deploy

Validate on real hardware

Load checkpoints, run them on robots, and push observed behavior back into the analysis loop.

InferenceLoad policies and run live inference on hardware.
VisualizeDebug failures through state, trajectory, and frame views.
Task OrchestrationDefine and replay repeatable task sequences.

Inside PRISM

Real deployment work also needs dataset editing, analysis, perception, and robot alignment. `prism_ai_tools` keeps those surrounding jobs inside the same app instead of pushing them into separate tools.

Data, Analysis, and Model Utilities

Data ToolsEdit datasets, merge episodes, run quality checks, and connect to Hugging Face.
AugmentExpand training data through vision and language augmentation.
VisualizeInspect state, trajectories, and frames from Parquet and HDF5 sources.
EmbeddingExtract checkpoint embeddings and interpret them with t-SNE or UMAP.

Perception and Robotics Operations

DetectionRun Grounding DINO object detection and save the result set.
SAM2 / RT-SAMCreate and track object masks offline or on live streams.
KinematicsConvert joint-based data into end-effector-space signals.
Camera CalibrationRealign extrinsics and recover from calibration drift.
