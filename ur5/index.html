Edit Dataset — 7 Tools

수집 후 데이터셋 품질 관리를 위한 7가지 도구. Quality Check → Delete → Filter → Merge 순서가 일반적입니다.

Quality Check

데이터 유효성 검증 및 품질 점수 산출. 누락 프레임, 이상치, 카메라 동기화 확인.

Filter Dataset

전문가 선별 및 카메라별 필터링. 특정 조건에 맞는 에피소드만 추출.

Upload & Download

HuggingFace Hub 업로드/다운로드. 팀 간 데이터셋 공유 및 버전 관리.

Merge Dataset

여러 세션에서 수집한 데이터셋을 하나로 병합. 인덱스 자동 재배열.

Delete Episodes

실패/불량 에피소드 삭제. annotation 결과(FAIL) 기반으로 빠르게 제거.

Change Task Instruction

태스크 지시문 일괄 수정. 오타 수정이나 더 구체적인 설명으로 변경.

Scatter Plot

관절 위치/속도 분포 시각화. 이상 에피소드를 시각적으로 탐색.

Data Augmentation

Vision과 Language 양쪽으로 데이터를 증강하여 학습 다양성을 확보합니다. Stream(실시간 미리보기)과 Batch(오프라인 일괄 적용) 모드 지원.

Vision Augmentation

Lightness ±100%

조명 변화 대응

Gaussian Noise σ 0 ~ 0.3

센서 노이즈 대응

Salt & Pepper 0 ~ 20%

픽셀 결함 대응

Motion Blur 0 ~ 10px

모션 블러 대응

Color Jitter Hue / Saturation

색상 변화 대응

Disturbance on / off

가림(occlusion) 대응

Language Augmentation

Manual

직접 지시문 변형 입력. 예: “컵을 집어라” → “빨간 컵을 테이블에서 집어라”. 에피소드별 랜덤 할당 가능.

LLM (EXAONE)

LLM으로 태스크 지시문 자동 생성. 다양한 표현을 자동으로 만들어 데이터 다양성 확보. (현재 비활성)

Mode Stream

실시간 카메라 피드에서 원본 vs 증강 비교 미리보기.

Mode Batch

저장된 데이터셋에 오프라인 일괄 적용. 증강 비율 설정 가능.

Output {dataset}_aug_{ts}/

증강된 데이터는 원본과 분리하여 별도 디렉터리에 저장.

End-to-End Workflow

UR5 파이프라인의 5단계 — 로봇 선택부터 증강된 학습 데이터 확보까지.

01 Select & Connect

Home에서 UR5 선택, ROS 연결 확인, Dashboard에서 Power On & Brake Release.

02 Verify Hardware

Devices에서 카메라, 관절 구조, URDF 3D 모델 확인. 관절 클릭으로 스펙 검증.

03 Collect Data

Record에서 텔레옵 수집. WARMUP → RECORDING → SAVING → RESET → ANNOTATE 순환.

04 Curate Dataset

Edit Dataset에서 Quality Check, 불량 에피소드 삭제, 필터링, 병합으로 데이터 정리.

05 Augment

Vision/Language 증강으로 학습 데이터 다양성 확보. Batch 모드로 일괄 적용.

Setup & Execution Order

데이터 수집(Record)까지의 전체 실행 순서입니다. 각 단계를 순서대로 실행하세요.

1
UR ROS 드라이버 실행

ros2 launch ur_robot_driver ur_control.launch.py ur_type:=ur5 robot_ip:=192.168.100.210
드라이버 연결 후 /joint_states, /tf 토픽 퍼블리시 시작.

2
UR 로봇 프로그램 실행

Teach Pendant에서 Remote Control 모드 확인 → External Control URCap 프로그램 로드 → Play.
또는 PRISM Devices 페이지의 Dashboard에서 Power On → Brake Release.

3
Controller Switching (Teleop 시)

ros2 control switch_controllers --deactivate scaled_joint_trajectory_controller --activate forward_velocity_controller
추론(Inference) 시에는 기본 scaled_joint_trajectory_controller 사용.

4
SpaceMouse Teleop 실행

SpaceMouse로 UR5를 실시간 조작. /forward_velocity_controller/commands로 velocity 명령 publish.
향후 리더 디바이스: PIKA Sense / Gripper / Station, PiPER 연동 예정.

5
RealSense Camera 실행

Wrist 카메라 (D405): ros2 launch realsense2_camera rs_launch.py camera_name:=cam_wrist camera_namespace:=camera
Third 카메라 (D435): ros2 launch realsense2_camera rs_launch.py camera_name:=cam_third camera_namespace:=camera
/compressed 토픽 없음 — image_transport republish 노드 필요.

6
PRISM 실행 (데이터 수집)

터미널 1: ros2 launch physical_ai_server physical_ai_server.launch.py
터미널 2: uvicorn ur5_server.main:app --host 0.0.0.0 --port 8001
브라우저에서 http://<PC IP>:3000 접속 → UR5 선택 → Record

Docker 빌드 & 실행

docker compose up -d — prism_manager (nginx :3000)와 prism_server (ROS 2 백엔드, network_mode: host) 2개 컨테이너.
컨테이너 내부: colcon build --packages-select physical_ai_server --symlink-install

FastAPI 단독 실행

pip install -e . 후 uvicorn ur5_server.main:app --host 0.0.0.0 --port 8001
Docker/ROS 2 없이도 동작. 로봇 연결 실패 시 health에서 connected: false 응답.

Service Init 9 Services

CatalogService → HFUser → CameraInventory → FileBrowse → RTDE → Dashboard → Safety → Gripper → StateBroadcaster

Streaming 125Hz → 30Hz

StateBroadcaster가 125Hz로 폴링, 30Hz로 WebSocket 브로드캐스트. 실시간 관절 모니터링.

Safety Max Speed Limits

Joint: 1.05 rad/s, TCP: 0.25 m/s. SafetyMonitor가 모션 전 안전 검증 수행.

Site Configuration — LG전자 생기원

현장 환경 설정 및 주의사항. 네트워크, 카메라, 컨트롤러, 조인트 매핑 정보.

Network 192.168.100.210

UR5 컨트롤러 IP. RTDE :30004, Dashboard :29999. ROS_DOMAIN_ID=30.

Robot ur_type: ur5

ur5e가 아님. 기본 컨트롤러: scaled_joint_trajectory_controller. SpaceMouse: forward_velocity_controller.

Camera RealSense D405 + D435

Wrist (D405) + Third-person (D435). 현재 PC USB 직접 연결. camera_name 파라미터 필수.

Gripper Robotiq 2F-85

finger_joint이 /joint_states에 없을 수 있음 — 6DoF로 동작 (missing joint skip).

Joint States 순서 주의

shoulder_lift_joint이 먼저, shoulder_pan_joint이 마지막. config의 joint_order로 이름 기반 매핑.

Teleop SpaceMouse (Velocity)

자체 좌표 변환 스크립트 사용. velocity 기반 제어. 향후 position 기반 전환 검토.

Leader (예정) PIKA & PiPER

Requirements Ubuntu 24.04 + ROS 2 Jazzy

Troubleshooting

현장에서 자주 발생하는 문제와 해결 방법.

카메라가 안 보임

ros2 topic list | grep camera로 토픽 확인. config의 camera_topic_list와 네임스페이스 일치 여부 확인. web_video_server 실행 상태 점검.

로봇이 안 움직임

ros2 control list_controllers로 활성 컨트롤러 확인. Teleop 시 forward_velocity_controller 활성화 필수. Teach Pendant에서 프로그램 실행 여부 점검.

PRISM UI 접속 안 됨

docker ps로 컨테이너 확인. curl localhost:3000 (프론트엔드), curl localhost:9090 (rosbridge) 점검. 방화벽 확인.

ROS_DOMAIN_ID 불일치

Docker compose에서 ROS_DOMAIN_ID=30 설정. 현장 PC에서 echo $ROS_DOMAIN_ID 확인 후 동일하게 맞춰야 통신 가능.

finger_joint 차원 불일치

현장 /joint_states에 finger_joint 없으면 6차원 텐서 생성. config joint_order는 7개 포함. data_converter가 skip하므로 6DoF 동작하나 학습/추론 시 차원 확인 필요.

컨트롤러 전환 누락

Teleop 전 forward_velocity_controller 활성화 잊으면 SpaceMouse 무반응. 추론 시 scaled_joint_trajectory_controller로 되돌려야 함.
